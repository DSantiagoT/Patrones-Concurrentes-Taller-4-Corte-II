# Patrones-Concurrentes-Taller-4-Corte-II
En el taller implemente las dos versiones de un sistema concurrente basado en el patron
productor-consumidor. La primera version emplea hilos tradicionales (threading) y la se-
gunda utiliza un ThreadPoolExecutor para gestionar la concurrencia de manera automatica.
En este documento analizo el flujo de trabajo, el rendimiento y la eficiencia de ambas ver-
siones, y reflexiono sobre la modularidad y escalabilidad del diseño.

# Ejecución
